# P11 全速处理与线程治理

## 目标
重构线程模型，保障全速流程可控（开始/进度/暂停预留/停止/异常恢复）。

## 本阶段新增文件
1. `infra/threading/task_runner.py`
2. `infra/threading/identify_task.py`
3. `infra/threading/slice_task.py`
4. `infra/threading/fullspeed_task.py`
5. `app/workflows/fullspeed_workflow.py`（增强）
6. `tests/integration/test_fullspeed_pipeline.py`
7. `tests/unit/test_task_runner.py`

## 本阶段迁移来源
- `cores/ThreadWorker.py`
- `ui/main_window.py`（全速相关交互）
- `ui/data_controller.py`（全速相关状态）

## 执行步骤
1. 线程职责收敛：任务只执行，不包含业务规则。
2. 用 workflow 编排任务：切片任务 -> 聚类识别任务 -> 保存任务。
3. 进度事件统一由 signal_bus 发出（分总进度/切片进度）。
4. 异常处理统一：任务抛错 -> workflow 捕获 -> UI 错误事件。
5. 预留暂停/继续接口（若本轮不实现，则保留明确状态机与 TODO）。
6. 处理资源释放：线程结束、模型句柄、临时文件。
7. 增加长任务回归测试与超时保护。

## 验收标准
1. 全速处理流程可稳定跑完。
2. 进度条、日志、结果输出一致。
3. 不再出现 UI 线程阻塞。

## 风险与应对
- 风险：跨线程数据竞争导致状态错乱。
- 应对：状态写入集中在 workflow，线程只返回结果。

## 回退点
- 线程调度层可降级到单线程顺序执行以排错。
