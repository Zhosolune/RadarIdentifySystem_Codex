# P01 工程骨架与入口

## 目标
建立可运行的 PyQt6 骨架，打通 FluentWindow，并初始化 `app/config.py`（QConfig）集中配置与页面样式分发链路。

## 本阶段新增文件
1. `main.py`
2. `app/__init__.py`
3. `app/application.py`
4. `app/config.py`
5. `app/logger.py`
6. `app/signal_bus.py`（最小壳）
7. `app/style_sheet.py`（页面样式枚举）
8. `app/resource_rc.py`（资源编译输出）
9. `ui/main_window.py`（FluentWindow 最小壳）
10. `resources/resource.qrc`（已接入资源编译）
11. `resources/qss/light/*.qss`
12. `resources/qss/dark/*.qss`
13. `utils/paths.py`
14. `tests/conftest.py`
15. `requirements.txt`

## 执行步骤
1. 初始化包结构（按 `docs/目录结构与分层约束.md`）。
2. 安装并验证 `PyQt6-Fluent-Widgets` 依赖。
3. 打通最小启动链路：`main.py -> app/application.py -> ui/main_window.py`。
4. 在 `app/config.py` 定义第一批 `ConfigItem` 默认值与校验。
   - 算法关键参数（clustering/identification/merge）
   - 绘图基础参数（`plot.scaleMode`、`plot.onlyShowIdentified`、`plot.order`）
5. 通过 qconfig 完成最小读写验证，持久化到 `config/config.json`。
6. 使用官方主题接口同步配置：`setTheme(appConfig.get(appConfig.themeMode))` 与 `setThemeColor(appConfig.get(appConfig.themeColor))`。
7. 将页面样式改为 `StyleSheet.<PAGE>.apply(widget)`，并支持“资源优先、本地回退”。
8. 接入资源编译链路：`resources/resource.qrc -> app/resource_rc.py`，并在启动时导入资源模块完成注册。
9. 单人开发模式：以 `python main.py` 作为唯一启动方式，不引入 scripts 启动脚本。

## 验收标准
1. 新目录可独立运行。
2. FluentWindow 能正常显示。
3. QConfig 可读取默认值并保存。
4. 主题/主题色在重启后可恢复。
5. `StyleSheet.WELCOME_PAGE.apply(self)` 可成功加载样式（优先编译资源，失败回退本地文件）。
6. 资源编译链路可用：`resources/resource.qrc -> app/resource_rc.py`。
