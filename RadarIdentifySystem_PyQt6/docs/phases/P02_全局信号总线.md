# P02 全局信号总线（app/signal_bus.py）

## 目标
建立统一跨模块事件机制，降低 UI、工作流、核心模块之间的直接耦合。

## 本阶段新增/修改文件
1. `runtime/events.py`
2. `app/signal_bus.py`
3. `tests/unit/test_signal_bus.py`

## 信号设计清单（第一版）
1. 数据导入：`data_import_started`、`data_import_finished`、`data_import_failed`
2. 切片：`slice_started`、`slice_ready`、`slice_changed`
3. 识别：`identify_started`、`identify_progress`、`cluster_ready`、`identify_finished`
4. 合并：`merge_started`、`merge_finished`
5. 导出：`export_started`、`export_progress`、`export_finished`、`export_failed`
6. 配置：`config_changed`、`theme_changed`
7. UI反馈：`toast_requested`、`error_reported`

## 事件模型（runtime/events.py）
1. `ImportFinishedEvent`
2. `SliceReadyEvent`
3. `IdentifyProgressEvent`
4. `ClusterReadyEvent`
5. `MergeFinishedEvent`
6. `ExportProgressEvent`
7. `ErrorEvent`

## 当前实现状态
1. 已完成事件模型与信号总线第一版代码。
2. 已补充单元测试文件 `tests/unit/test_signal_bus.py`。
3. 本地环境缺少 `pytest`，已使用等价内联断言脚本完成信号发射回归验证。

## 验收标准
1. 文档中的第一版信号在 `AppSignalBus` 中可见。
2. 事件对象可通过 `pyqtSignal(object)` 正确发射并被接收。
3. 后续模块统一通过 `signal_bus` 进行跨模块广播。
