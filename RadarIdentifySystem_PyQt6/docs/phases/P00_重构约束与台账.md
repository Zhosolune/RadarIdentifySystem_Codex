# P00 重构约束与台账初始化

## 目标
在编码前冻结重构边界、验收规则、迁移台账，避免后期返工。

## 输入
- `docs/legacy_full_file_inventory.txt`
- `docs/legacy_python_file_inventory.txt`
- 旧项目现状分析结论
- `docs/PyQtFluentWidgets可行性评估.md`

## 本阶段产出文件
1. `docs/重构阶段索引.md`
2. `docs/PyQt6重构总体规划.md`
3. `docs/目录结构与分层约束.md`
4. `docs/配置系统设计.md`
5. `docs/功能对齐矩阵.md`
6. `docs/迁移台账.csv`
7. `docs/风险清单.md`
8. `docs/重构接口对接手册.md`
9. `docs/重构执行追踪.md`

## 执行步骤
1. 建立“功能对齐矩阵”。
2. 建立“迁移台账”。
3. 固定目录基线：采纳 easyVer 风格，并新增 `infra` 层。
4. 固定配置系统方案：全部采用 QConfig（`app/app_config.py` + `config/config.json`）。
5. 固定线程归属：`workers` 统一落在 `runtime/threading`，不放 `core`。
6. 固定“不做项”：本轮不做功能新增，仅补 MAT 解析占位标准接口。
7. 固定质量门禁与回退策略。
8. 固定数据基线与性能基线。
9. 确认 Fluent 许可策略（是否涉及商用分发）。

## 验收标准
1. 全量文件都在迁移台账中有记录。
2. 每个功能点都能在功能对齐矩阵中找到承接模块。
3. 配置系统与 UI 框架策略一致。
4. 目录结构和分层约束有明确结论。
5. 许可风险有明确结论。
