# 配置系统设计（QConfig 单一方案）

## 1. 目标
全部采用 Fluent `QConfig` 作为唯一配置源，并按官方最佳实践在 `app/app_config.py` 中集中管理所有配置项。

## 2. 设计原则
1. 单一配置源：仅 `QConfig`，不维护第二套业务配置系统。
2. 集中定义：所有配置项在 `app/app_config.py` 定义。
3. 强校验：每个 `ConfigItem` 绑定 `ConfigValidator`。
4. 可序列化：通过 `qconfig.load("config/config.json", appConfig)` 加载，使用 `appConfig.save()` 持久化到 `config/config.json`。
5. 主题配置遵循官方分组：`QFluentWidgets.ThemeMode` 与 `QFluentWidgets.ThemeColor`。
6. 结果可复现：影响算法行为与绘图结果的参数都必须可持久化。

## 3. 推荐结构（app/app_config.py）
1. `class AppConfig(QConfig):`
   - `QFluentWidgets.ThemeMode`（继承自 `QConfig.themeMode`）
   - `QFluentWidgets.ThemeColor`（继承自 `QConfig.themeColor`）
   - `algorithm.clustering.*`
   - `algorithm.identification.*`
   - `algorithm.extraction.*`
   - `merge.*`
   - `export.*`
   - `runtime.*`
   - `plot.*`

2. 文件底部创建单例：
   - `appConfig = AppConfig()`
   - `qconfig.load("config/config.json", appConfig)`
   - `setTheme(appConfig.get(appConfig.themeMode), save=False)`
   - `setThemeColor(appConfig.get(appConfig.themeColor), save=False)`

## 4. plot.* 必配字段
1. `plot.scaleMode`：`STRETCH / STRETCH_BILINEAR / STRETCH_NEAREST_PRESERVE`
2. `plot.onlyShowIdentified`：是否仅展示识别后目标
3. `plot.order`：图顺序（默认 `CF,PW,PA,DTOA,DOA`）
4. `plot.dimension.<DIM>.yMin / yMax / imgHeight / imgWidth`
5. `plot.bandProfile.<Band>.cfMin / cfMax`
6. `plot.bandProfile.X.overridePwMax / overrideDtoaMax`（X 波段特殊覆盖）
7. `plot.mergePalette`：合并聚类颜色表（保证跨环境一致）

说明：
- 自动检测得到的“当前波段结果”属于运行时状态，不持久化。
- 频段策略阈值、维度范围属于配置，必须持久化。

## 5. 配置使用方式
1. 读取：`appConfig.get(item)` 或 `item.value`
2. 写入：`qconfig.set(item, value)` （注意：严禁直接使用 `item.value = value`，否则不会触发序列化与信号）
3. 保存：`appConfig.save()`
4. 主题应用：`setTheme(...)`、`setThemeColor(...)`
5. 指纹：对 `appConfig.toDict()` 排序后哈希

## 6. 是否按任务拆参数类
1. 持久化层：不拆多个 `QConfig`，统一 `AppConfig`。
2. 业务层：可按任务拆轻量参数对象（如 `ClusterParams`、`PlotParams`），由 `AppConfig` 映射生成。
3. 这样既保持配置中心统一，又保证算法与 UI 调用解耦。
4. 具体规则以 `docs/算法参数对象规则.md` 为准。

## 7. 迁移策略
1. 旧 `default_config` 字段映射到 `AppConfig`。
2. 先迁移算法关键参数与 plot 基础参数，再迁移完整 `plot.dimension` 与 `plot.bandProfile`。
3. 删除 `ConfigService` 路线，避免双配置。

## 8. 维护规则
1. 新增配置项必须先在 `app/app_config.py` 定义默认值与校验。
2. 配置项变更必须同步更新：
   - `docs/重构接口对接手册.md`
   - `docs/重构执行追踪.md`
3. 不允许在业务代码里硬编码参数常量（测试代码除外）。
