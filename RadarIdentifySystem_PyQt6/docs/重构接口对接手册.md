# 重构接口对接手册

## 1. 目的
记录重构后模块接口含义和用法，避免对接时凭空猜测。

## 2. 使用规则
1. 接口变更先改文档再改代码。
2. 模块内调用优先函数，不滥用 signal_bus。
3. 大型 `np.ndarray` 不直接走全局信号。
4. 配置字段变更同步更新 `docs/配置系统设计.md`。
5. 目录结构调整同步更新 `docs/目录结构与分层约束.md`。

## 3. 关键接口

### 3.0 main.py
- `main() -> int`
- 责任：单人开发模式下的唯一应用启动入口（`python main.py`）。

### 3.1 app/signal_bus.py
- `data_import_started(path: str)`
- `data_import_finished(event: ImportFinishedEvent)`
- `data_import_failed(event: ErrorEvent)`
- `slice_started()`
- `slice_ready(event: SliceReadyEvent)`
- `slice_changed(slice_idx: int)`
- `identify_started()`
- `identify_progress(event: IdentifyProgressEvent)`
- `cluster_ready(event: ClusterReadyEvent)`
- `identify_finished(success: bool, cluster_count: int)`
- `merge_started()`
- `merge_finished(event: MergeFinishedEvent)`
- `export_started(path: str)`
- `export_progress(event: ExportProgressEvent)`
- `export_finished(path: str)`
- `export_failed(event: ErrorEvent)`
- `error_reported(event: ErrorEvent)`
- `config_changed(key: str, value: object)`
- `theme_changed(mode: str, color: str)`
- `toast_requested(level: str, message: str)`

### 3.2 runtime/events.py
- `ImportFinishedEvent(file_path, source_type, total_pulses, filtered_pulses, band)`
- `SliceReadyEvent(slice_count, time_ranges)`
- `IdentifyProgressEvent(current_slice_idx, total_slices, progress)`
- `ClusterReadyEvent(slice_idx, cluster_idx, dim_name, is_valid)`
- `MergeFinishedEvent(slice_idx, merged_count)`
- `ExportProgressEvent(progress, message)`
- `ErrorEvent(code, message, detail, trace_id)`

### 3.3 app/application.py
- `create_qapplication(argv: Sequence[str] | None = None) -> QApplication`
- `create_main_window() -> MainWindow`
- `run(argv: Sequence[str] | None = None) -> int`

### 3.4 core/data
- `preprocess(raw_data: np.ndarray) -> PreprocessResult`
- `slice_by_toa(data: np.ndarray, slice_length_ms: float) -> SliceResult`

### 3.5 core/clustering
- `run_clustering(slice_data: np.ndarray, params: ClusterParams) -> ClusteringResult`

### 3.6 core/recognition
- `evaluate_cluster(cluster: ClusterData, pred: RawPrediction, params: IdentifyParams) -> PredictionResult`

### 3.7 core/merge
- `merge_clusters(clusters: list[ClusterFeature], params: MergeParams) -> MergeResult`

### 3.8 infra/parsers
- `parse_excel(path: str) -> np.ndarray`
- `parse_bin(path: str, strategy: str) -> np.ndarray`
- `parse_mat(path: str) -> np.ndarray`（未实现阶段返回标准错误）

### 3.9 app/app_config.py（单一配置入口）
- `class AppConfig(QConfig)`：集中定义全部配置项与默认值。
- `appConfig`：全局配置实例。
- `qconfig.load("config/config.json", appConfig)`：加载配置。
- `appConfig.save()`：保存配置。

说明：
- 不再使用 `ConfigService`。
- 所有配置都从 `appConfig` 读取。

### 3.10 配置分组对接约定
1. `algorithm.*`：聚类/识别/参数提取所需阈值与权重。
2. `merge.*`：三类合并规则阈值。
3. `export.*`：导出路径与切分策略。
4. `plot.*`：绘图相关配置（必须持久化）。

`plot.*` 最小对接字段：
- `plot.scaleMode`
- `plot.onlyShowIdentified`
- `plot.order`
- `plot.dimension.*`
- `plot.bandProfile.*`
- `plot.mergePalette`
