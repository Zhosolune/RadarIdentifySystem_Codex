## 2. 分层职责
1. `app`：生命周期、配置入口、全局单例、信号总线。
2. `core`：纯业务规则与算法，不依赖 UI/线程/数据库。
3. `infra`：第三方/系统适配（文件解析、推理、线程、导出、绘图）。
4. `ui`：界面组件与交互，不直接承载业务算法。
5. `utils`：无业务语义的通用工具。

## 3. 强制依赖方向
- `ui -> app -> core`
- `app -> infra`
- `infra -> core`（仅调用 core 暴露能力）
- `core` 禁止依赖 `ui/app/infra`

## 4. 明确约束
1. `workers` 不得放入 `core`，统一落在 `infra/threading`。
2. 配置唯一入口为 `app/config.py`，持久化文件为 `config/config.json`。
3. 不允许新增第二套配置系统。
4. 不允许在 `ui` 直接调用 `core`/`infra` 具体实现。
