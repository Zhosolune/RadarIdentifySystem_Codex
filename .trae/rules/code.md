# 代码规范
## Python
- 编写Python函数时，遵循以下规则：
  - **格式要求**：使用标准的Python docstring格式（三重引号）和 Google 风格的注释文档
  - **内容要求**：
    - 每个有输入参数（self除外）的函数/方法的文档注释必须包含：
      - 功能描述：简洁明确地说明该函数的作用和目的
      - Args：列出所有输入参数，包括参数名、类型、含义和默认值（如有）
      - Returns：描述返回值的类型和含义
      - Raises：列出可能抛出的异常类型和触发条件
    - 要为函数内代码逻辑编写必要的行内注释，尤其是关键操代码，要注明其功能、作用或目的，注释均使用简体中文，且简洁精炼
- 编写Python类时，遵循以下规则：
  - **格式要求**：使用标准的Python docstring格式（三重引号）和 Google 风格的注释文档
  - **内容要求**：
    - 每个类的文档注释必须包含：
      - 功能描述：简洁明确地说明该类的功能
      - Attributes：列出所有属性，包括属性名、类型、含义和默认值（如有）
    - 要为类内代码逻辑编写必要的行内注释，尤其是关键操代码，要注明其功能、作用或目的，注释均使用简体中文，且简洁精炼
- 注意检查代码质量：
    - 变量类型不匹配
    - 函数返回值类型检查
    - 函数参数类型检查
    - 类属性类型检查
    - 版本弃用类型检查
    - 类型注释完整性检查
